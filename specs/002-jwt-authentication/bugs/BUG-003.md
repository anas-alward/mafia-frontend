# Bug Report: Email Verification After Signup + Login with Email

**Type**: Spec gap
**Severity**: High
**Feature**: 002-jwt-authentication
**Reported**: 2026-06-26

## Description

Two intertwined changes to the authentication flow:

1. **Email verification after signup**: After submitting the sign-up form successfully, the user stays on the same page and is presented with an email verification code input field. They enter the code and the frontend sends it to `POST /accounts/verify-email/`. Only after successful verification are they authenticated and redirected to the dashboard.

2. **Login with email instead of username**: The login form and its backend request must use the email field instead of username. The Django backend's `/accounts/login/` endpoint now expects `email` + `password`.

## Root Cause Analysis

This is a **spec gap** — the original specification explicitly assumed "No email verification is required for v1" and used username-based login. The Django backend has evolved to require email verification after signup and email-based login, but the frontend specification was never updated to match. The user is now surfacing this as a bug that needs to be reconciled across all artifacts.
