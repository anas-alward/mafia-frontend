# Bug Report: Wrong Color Palette — Dark Instead of Minimalist Light

**Type**: Spec conflict
**Severity**: High
**Feature**: 001-mafia-game-landing
**Reported**: 2026-06-26

## Description

The landing page uses a dark atmospheric color scheme (dark backgrounds, teal/lagoon accents, radial gradients) based on an incorrect assumption during specification. The user explicitly requested a **minimalist** design with:

- **Main background**: White
- **Action color**: Black or shades of black
- **Overall feel**: Clean, minimal, professional — NOT dark/gamified atmospheric

The current implementation uses `<html class="dark">`, a dark color palette (`--bg-base: #0a1418`), teal CTAs (`--lagoon: #60d7cf`), and complex gradient backgrounds — all of which violate the user's intent for a clean, white, minimalist aesthetic.

## Root Cause Analysis

During specification (`/speckit-specify`), the user's description included the word "gamified" alongside "minimalist." The spec author interpreted "gamified" as dark/atmospheric (common in game websites like Valorant, League of Legends) and made it the default assumption. The user's repeated emphasis on "minimalist" and "pro ui ux" should have steered toward a clean, light aesthetic (think Linear.app, Stripe, or Notion — professional, white backgrounds, black/dark accents).

The assumption was not flagged as [NEEDS CLARIFICATION] because it seemed reasonable at the time. This is a case where a reasonable default was wrong for this user's specific taste.

## Recommended Fix

1. Run `/speckit.bugfix.patch` to update spec assumptions (dark → light), research R4 (reverse dark mode decision), and plan color direction
2. Run `/speckit.bugfix.verify` to confirm artifact consistency
3. Re-implement the color palette:
   - Switch to light mode default (`<html>` without `dark` class, or light theme)
   - Set background to white/near-white
   - Set action/CTA colors to black or dark gray shades
   - Strip or drastically simplify the atmospheric gradients in `styles.css`
   - Keep shadcn zinc base tokens (already minimalist)
   - Remove custom color tokens (`--lagoon`, `--sea-ink`, `--palm`, etc.) or repurpose as subtle accent only
